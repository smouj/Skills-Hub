# SKILL Template (smouj standard)

Use this template when creating new skills to keep consistent structure.

## Required sections
1. Frontmatter (name, description, metadata.openclaw)
2. Purpose
3. Scope
4. Inputs expected
5. Outputs
6. Standard workflow (5 steps)
7. Commands
8. Guardrails
9. Installation notes
10. Test checklist
11. Changelog

## Quality rules
- Clear purpose in one paragraph.
- Explicit guardrails (secrets, destructive ops).
- Reproducible verification checks.
- Rollback note when applicable.
- Concise, operational language.
